Payment Provider Checkout URL Extractor - RESTful API

Quick Start

Extract checkout URLs and payment providers from e-commerce websites with a simple API call.

# Direct API call with URL parameter
curl "https://payment-provider-extractor.vercel.app/api/url/https%3A//example-shop.com"

# Check status
curl "https://payment-provider-extractor.vercel.app/api/session/YOUR_SESSION_ID"

Direct API Endpoint

GET /api/url/:encoded_url

Start analysis and get complete results immediately by providing the website URL directly in the path (URL-encoded). This endpoint runs synchronously and returns the full analysis results.

GET/api/url/{encoded_website_url}

Parameters:

  • encoded_url (path parameter) - The e-commerce website URL to analyze (must be URL-encoded)

Example URLs:

https://payment-provider-extractor.vercel.app/api/url/https%3A//demo.shopify.com
https://payment-provider-extractor.vercel.app/api/url/https%3A//example-shop.com
https://payment-provider-extractor.vercel.app/api/url/https%3A//store.example.com

POST API Endpoint

POST /api/analyze

Start analysis by sending a POST request with JSON body.

POST/api/analyze

Request Body:

{
  "url": "https://example-shop.com",
  "detailed": true
}

Parameters:

  • url (required) - The e-commerce website URL to analyze
  • detailed (optional) - Include detailed analysis (default: true)

Features

  • Automated e-commerce checkout analysis
  • Payment provider identification
  • Checkout URL extraction
  • Multilingual support (English, French, Italian, German, Spanish, Polish)
  • Aggressive popup handling and elimination
  • 2-minute timeout protection
  • Live session URLs for real-time viewing
  • Real-time progress updates
  • RESTful API design
  • JSON responses

Response Times

  • Target execution time: 5-10 seconds
  • Maximum timeout: 2 minutes
  • Typical response: 10-30 seconds
  • Complex sites: 30-60 seconds

Status Codes

  • 200 Success
  • 400 Bad Request (invalid URL, missing parameters)
  • 404 Not Found (session not found)
  • 500 Internal Server Error

üí° Usage Tips

  • URLs in the direct API endpoint should be URL-encoded
  • Use the session ID to check progress and get results
  • Analysis runs in the background - check status periodically
  • Results are available for 1 hour after completion
  • For real-time updates, use the web interface with Socket.io

üîß Integration Examples

JavaScript/Fetch:

const url = encodeURIComponent('https://example-shop.com');
const response = await fetch(`https://payment-provider-extractor.vercel.app/api/url/${url}`);
const data = await response.json();
console.log('Session ID:', data.session_id);

Python/Requests:

import requests
from urllib.parse import quote
url = quote('https://example-shop.com', safe='')
response = requests.get(f'https://payment-provider-extractor.vercel.app/api/url/{url}')
data = response.json()
print('Session ID:', data['session_id'])
